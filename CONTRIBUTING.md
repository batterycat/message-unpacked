# Contributing

Thank you for helping students make safer decisions online.

1. Read `CONSTITUTION.md`.
2. Open an issue before adding a new dependency or substantial case family.
3. Create cases from `content/cases/_template.yaml` and validate them locally.
   Follow `docs/CONTENT_AUTHORING.md` and `docs/CASE_REVIEW_CHECKLIST.md`.
4. Record every external dependency, source, quotation, dataset, or media asset
   in `docs/THIRD_PARTY_RESOURCES.md`.
5. Run the complete quality checks listed in `README.md`.

For classroom-use feedback, consult the public
[Teacher Guide](https://loomtype.gitbook.io/loomtype-docs/) and report unclear
instructions or teaching-flow issues in GitHub Issues. Keep factual case
changes and source-backed claims in the repository so they remain reviewable in
pull requests.

Case contributions must use respectful language, remove personal information,
distinguish provenance accurately, and qualify every impact number. A source
citation alone does not permit copying source text or media.

By intentionally submitting code, you agree that the contribution is provided
under Apache-2.0. By intentionally submitting original educational content, you
agree that it is provided under CC BY-SA 4.0. Do not submit material you do not
have the right to license on those terms.
